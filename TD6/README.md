# Phase 4 - RAG with Knowledge Graphs

Retrieval-Augmented Generation (RAG) using the RDF knowledge base and a local small LLM. Converts natural language questions to SPARQL, executes on the graph, and presents grounded results.

## Prerequisites

- Python ≥ 3.9
- `rdflib`, `requests` (see project `requirements.txt`)
- **Ollama** installed and running locally
- Knowledge base: `../livrable/kb_expanded.nt` (generated by Phase 2 pipeline)

## Setup

1. **Install dependencies** (from project root):
   ```bash
   pip install -r requirements.txt
   ```

2. **Start Ollama** and run a model:
   ```bash
   ollama run gemma:2b
   ```
   Alternatives: `qwen3.5:0.8b`, `llama3.2:1b`, `deepseek-r1:1.5b`

3. **Verify Ollama** is running:
   ```bash
   curl http://localhost:11434/api/generate -d '{"model":"gemma:2b","prompt":"who are you?","format":"json"}'
   ```
   You should see a JSON response with a `response` field.

## How to Run

From the **project root**:

```bash
python TD6/lab_rag_sparql_gen.py
```

Or from the TD6 directory:

```bash
cd TD6
python lab_rag_sparql_gen.py
```
(Ensure `../livrable/kb_expanded.nt` exists.)

## Usage

The script runs an interactive CLI. For each question:

1. **Baseline (No RAG)**: The LLM answers directly without the knowledge graph.
2. **SPARQL-generation RAG**: The LLM generates SPARQL from your question, the query is executed on the KB, and results are displayed.

Type `quit` to exit.

## Example Questions (Local History domain)

- What places (spatial) are associated with Bucarest?
- How many Europeana items have subject "archaeology"?
- What entity types (Person, Place, Organization) exist in the KB?
- Which items are linked to Romania?
- What creators (dc:creator) appear in the base?

## Troubleshooting

- **"Cannot connect to Ollama"**: Start Ollama (`ollama run gemma:2b`) before running the script.
- **"KB file not found"**: Run the Phase 2 pipeline first to generate `livrable/kb_expanded.nt`.
- **"model not found"**: Pull the model with `ollama pull gemma:2b` or use another model by editing `GEMMA_MODEL` in the script.
- **"Ollama timed out"**: SPARQL generation with a large schema can take several minutes. Increase `OLLAMA_TIMEOUT` (default 600s) in the script, or use a faster/smaller model such as `qwen3.5:0.8b`.
